SKD International — Getting started as a team member

Access Levels

SKD has two roles. Your access depends on which one you are.

Admin

Full access to everything — VPS root, GitHub admin, OANDA keys, infrastructure decisions. Currently: Niksa + xxtox1chxx only.

Member

Access to shared knowledge, Discord, GitHub repos (read + PR), Obsidian vault. No VPS root, no trading keys, no infra changes.

3 GitHub & SSH

Set up Git and clone the repos you have access to.

# Generate SSH key (if you don't have one)
ssh-keygen -t ed25519 -C "your-email@example.com"

# Copy your public key
# Mac:
pbcopy < ~/.ssh/id_ed25519.pub
# Windows:
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard

# Add to GitHub: Settings → SSH Keys → New SSH Key → Paste

# Authenticate
gh auth login

# Clone repos
mkdir -p ~/Documents/skd
cd ~/Documents/skd
gh repo clone SKD-international/brain
gh repo clone SKD-international/docs
You can clone trading-infra and oanda-mcp too (read access), but you'll need to open a Pull Request for changes — you can't push to main directly.

4 Obsidian (BrainVault)

Open the shared knowledge base and set up auto-sync.

# Open Obsidian
# → "Open folder as vault"
# → Select: ~/Documents/skd/brain

# Set up auto-sync:
# 1. Settings → Community Plugins → Turn on community plugins
# 2. Browse → search "Obsidian Git" → Install → Enable
# 3. Obsidian Git settings:
#    - Auto pull interval: 5 minutes
#    - Auto push interval: 10 minutes
#    - Pull on startup: ON

Start with the Home note — it links to everything: trading docs, infra plans, learning tracks, templates.

5 Contributing

How to contribute as a member.

Obsidian / Brain Vault

Edit and add notes freely — Obsidian Git will auto-push your changes. Use the templates in 07 - Templates/ for consistency.

Code (trading-infra, oanda-mcp)

# Create a branch for your work
git checkout -b feature/your-feature-name

# Make changes, commit
git add .
git commit -m "Add your feature"

# Push and create a Pull Request
git push -u origin feature/your-feature-name
gh pr create --title "Your feature" --body "Description"

# An admin will review and merge
Do not push directly to main on code repos. Always use a branch + PR. Admins will review and merge.
