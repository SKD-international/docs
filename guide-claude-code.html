What is Claude Code?

Start Here

Your AI coding partner, right in the terminal

Claude Code is Anthropic's official command-line tool that brings Claude directly into your development workflow. Think of it as a brilliant colleague who can:

  • Read and write files across your entire codebase
  • Run terminal commands — build, test, deploy, anything
  • Search your code intelligently to understand context
  • Browse the web for documentation and references
  • Connect to external services via MCP servers (Slack, Notion, GitHub, and more)
  • Spawn sub-agents to handle parallel tasks simultaneously

💬 Natural Language

Just describe what you want in plain English. No special syntax required.

📁 Full File Access

Claude reads your project structure, understands dependencies, and modifies files directly.

🔌 Plugin Ecosystem

MCP servers let Claude interact with Slack, GitHub, Notion, your browser, and more.

📦

Installation

Required

Step 1 — Install via npm

You need Node.js 18+ installed. Then run:

Terminal 
$ npm install -g @anthropic-ai/claude-code

Step 2 — Launch Claude Code

Navigate to your project directory and start Claude:

Terminal 
$ cd my-project
$ claude

Claude will detect your project, read your codebase, and be ready for instructions.

Step 3 — Authenticate

On first launch, you will be prompted to log in with your Anthropic account. Follow the browser-based auth flow, and you are set.

💡
Tip: Create a CLAUDE.md file in your project root with instructions for Claude. It reads this automatically on every conversation start.
⌨️

Basic Usage

Required

Just type what you want

Claude Code understands natural language. There is no special syntax to learn. Simply describe what you need:

You type 
> fix the bug in auth.py where users can't log in with email

Claude will:

  1. Search for auth.py in your project
  2. Read the file and understand the authentication logic
  3. Identify the bug and propose a fix
  4. Apply the fix directly to the file (with your approval)

🐛 Fix Bugs

> fix the TypeError in utils/parser.js

✨ Add Features

> add a login page with email and password fields

📖 Explain Code

> explain what the process_payment function does

🧪 Write Tests

> write unit tests for the User model

♻️ Refactor

> refactor the database module to use async/await

🚀 Deploy

> build the project and deploy to production

Slash Commands

Essential

Slash commands are shortcuts that trigger specific behaviors. Type them directly into the Claude Code prompt.

CommandWhat It DoesWhen To Use
/helpShow available commands and usage infoWhen you are lost or need guidance
/clearClear the current conversationStarting fresh on a new task
/compactCompress conversation contextWhen the conversation gets long and slow
/reviewReview your code changesBefore committing, get AI feedback
/commitCommit with an AI-generated messageAfter making changes you are happy with
/specify [prompt]Create a BrainGrid requirement specTurning an idea into a structured plan
/build [req-id]Get implementation plan for a requirementReady to start building a specified feature
Example: Specify → Build workflow
Step 1: Create a requirement 
> /specify "Add user authentication with OAuth2 support"
✓ Created REQ-042: User Authentication with OAuth2
  - Email/password login
  - Google OAuth2 integration
  - JWT token management
  - Session handling
Step 2: Get the build plan 
> /build REQ-042
✓ Implementation plan for REQ-042:
  Phase 1: Database schema (User, Session tables)
  Phase 2: Auth routes (/login, /register, /oauth/google)
  Phase 3: JWT middleware
  Phase 4: Frontend login form
  Phase 5: Tests

MCP Servers

Power Feature

What is MCP?

Model Context Protocol (MCP) is a plugin system that lets Claude talk to external services. Think of MCP servers as bridges — each one connects Claude to a different tool or platform. When an MCP server is active, you can simply ask Claude to interact with that service in plain English.

✅
No API keys needed on your part. MCP servers handle authentication. Just ask Claude to do things and it routes the request to the right service.

Skills / Superpowers

Productivity

Skills are reusable AI workflows — specialized modes that Claude can enter to tackle tasks with a structured approach. Think of them as expert playbooks.

💡 Brainstorming

Explore ideas and approaches before writing any code. Great for architecture decisions.

📋 Writing Plans

Create detailed implementation plans with phases, dependencies, and estimated effort.

🏗️ Executing Plans

Step-by-step execution of implementation plans, checking off tasks as they complete.

🧪 Test-Driven Development

Write tests first, then implement code to make them pass. Red-green-refactor cycle.

🔍 Systematic Debugging

Methodical approach: reproduce, isolate, hypothesize, test, fix, verify.

🔎 Code Review

Review pull requests for bugs, performance, security, and style issues.

🚀 Feature Development

Guided end-to-end feature development from spec to implementation to tests.

🎨 Frontend Design

Create beautiful, responsive user interfaces with modern CSS and frameworks.

📄 Office Files

Work with PDF, XLSX, DOCX, and PPTX files. Read, create, and manipulate documents.

Agent System

Advanced

Claude Code can spawn sub-agents — independent AI instances that work on specific sub-tasks in parallel. This is like having a team of specialists working simultaneously.

🔍 Explorer Agent

Searches and analyzes the codebase to answer questions about architecture, patterns, and dependencies.

// Claude automatically spawns this when
// researching a large codebase
> how is authentication handled across the app?

📐 Plan Agent

Creates architecture plans and implementation strategies based on codebase analysis.

> plan how to add real-time notifications

📝 Review Agent

Reviews code changes for correctness, performance, security, and best practices.

> /review

⚡ Parallel Execution

Multiple agents work at the same time. While one researches the database schema, another can analyze the API routes.

💡
You do not need to manually spawn agents. Claude automatically decides when to use sub-agents based on the complexity of your request.

Tips & Tricks

Level Up

💾 Use CLAUDE.md

Put a CLAUDE.md file in your project root. Claude reads it at the start of every conversation. Include coding standards, architecture notes, and preferred patterns.

🧹 Use /compact

When conversations get long, Claude's context window fills up and responses slow down. Run /compact to compress the history while keeping key information.

🎯 Be Specific

Instead of "fix the bug", say "fix the null pointer exception in auth.py line 42 that occurs when email is empty". More context = better results.

🔄 Iterative Approach

Build features incrementally. Ask Claude for small steps, review each one, then continue. This gives you more control over the output.

🧪 Let Claude Test

Claude can run your test suite, see failures, and fix them. Say "run the tests and fix whatever fails" for a powerful feedback loop.

📓 Knowledge Management

Use Obsidian + Claude for a second brain. Claude can read, write, and search your notes, creating a persistent knowledge base across sessions.

Pro Tip: CLAUDE.md example 
# CLAUDE.md

## Project: My Trading App
## Stack: Python 3.12, FastAPI, PostgreSQL, React 19

## Coding Standards
- Use type hints everywhere
- Async by default
- Tests in /tests, mirror src structure
- Use Ruff for linting

## Architecture
- /src/api     - FastAPI routes
- /src/core    - Business logic
- /src/models  - SQLAlchemy models
- /src/services - External service integrations

## Important Notes
- Never commit .env files
- Always run tests before committing
- Use conventional commits
