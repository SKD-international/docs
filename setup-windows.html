Windows Setup Guide

SKD International — Development Environment Setup for Windows 10/11

1 Windows Terminal & Package Manager

Install Windows Terminal from the Microsoft Store, then set up winget (built into Windows 11) or install Scoop.

Open all terminals as Administrator for installation steps.
# Option A: winget (built-in on Windows 11)
winget --version

# Option B: Scoop (if winget unavailable)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Invoke-RestMethod -Uri https://get.scoop.sh | Invoke-Expression

2 Core Tools

Install Git, Python, Node.js, and Docker.

# Using winget
winget install Git.Git
winget install Python.Python.3.12
winget install OpenJS.NodeJS.LTS
winget install Docker.DockerDesktop
winget install GitHub.cli

# Restart terminal after installations

# Verify
git --version
python --version
node --version
docker --version
gh --version

3 SSH Key & GitHub

Generate an SSH key and connect to GitHub.

# Generate SSH key
ssh-keygen -t ed25519 -C "your-email@example.com"

# Start SSH agent
Get-Service ssh-agent | Set-Service -StartupType Automatic
Start-Service ssh-agent
ssh-add ~/.ssh/id_ed25519

# Copy public key to clipboard
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard

# Add to GitHub: Settings → SSH Keys → New SSH Key → Paste

# Authenticate with GitHub CLI
gh auth login

# Clone the org repos
mkdir ~/Documents/skd
cd ~/Documents/skd
gh repo clone SKD-international/brain
gh repo clone SKD-international/trading-infra
gh repo clone SKD-international/oanda-mcp
gh repo clone SKD-international/docs

7 WSL2 (Required for Claude Code)

Claude Code does not run natively on Windows. You need WSL2 (Windows Subsystem for Linux) — it gives you a real Ubuntu environment inside Windows. required

This is not optional if you want to use Claude Code. It's also recommended for Python/Docker development parity with the Hetzner server.
# Step 1: Install WSL2 (run in PowerShell as Admin)
wsl --install

# This installs Ubuntu by default. RESTART your PC when prompted.

# Step 2: After restart, Ubuntu opens automatically.
# Create a username and password when prompted.

# Step 3: Update Ubuntu
sudo apt update && sudo apt upgrade -y

# Step 4: Install Node.js (required for Claude Code)
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# Step 5: Install Python and other tools
sudo apt install -y python3 python3-pip python3-venv git redis-tools

# Step 6: Set up SSH key inside WSL
ssh-keygen -t ed25519 -C "your-email@example.com"
cat ~/.ssh/id_ed25519.pub
# Copy the output and add it to GitHub (Settings → SSH Keys)

# Step 7: Clone repos inside WSL
mkdir -p ~/skd
cd ~/skd
git clone git@github.com:SKD-international/brain.git
git clone git@github.com:SKD-international/trading-infra.git
git clone git@github.com:SKD-international/oanda-mcp.git
git clone git@github.com:SKD-international/docs.git
Tip: In Windows Terminal, click the dropdown arrow → you'll see "Ubuntu" as an option. Use that for all dev work. You can also open WSL from any folder in File Explorer by typing wsl in the address bar.
VS Code integration: Install the "WSL" extension in VS Code. Then from your WSL terminal, type code . to open any folder in VS Code with full Linux support.

8 Claude Code (inside WSL)

Install and run Claude Code inside your WSL2 Ubuntu environment. required

# Make sure you're in WSL (Ubuntu terminal, NOT PowerShell)

# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Verify installation
claude --version

# Run in any project directory
cd ~/skd/trading-infra
claude

# First time: it will ask you to authenticate with your Anthropic account
# Follow the browser link to log in
Common mistake: Do NOT install Claude Code in PowerShell or CMD. It must be installed and run inside WSL2. If you see errors about "unsupported platform", you're in the wrong terminal.
Access Windows files from WSL: Your Windows drives are mounted at /mnt/c/, /mnt/d/ etc. But for best performance, keep your code inside the WSL filesystem (~/skd/), not on /mnt/c/.

9 Docker Desktop + WSL2

Docker Desktop needs to be configured to use WSL2 as its backend.

# Docker Desktop should already be installed from Step 2
# Open Docker Desktop → Settings → General
# ✅ "Use the WSL 2 based engine" should be checked

# Settings → Resources → WSL Integration
# ✅ Enable integration with your Ubuntu distro

# Verify Docker works inside WSL:
docker run hello-world
Once configured, docker and docker compose commands work seamlessly inside WSL. This is how you'll run n8n, Redis, and other containers locally.

10 Hetzner VPS Access

Connect to the shared production server. Use WSL terminal for SSH.

# From inside WSL — your SSH key is already set up from Step 7
# Send your public key to Niksa:
cat ~/.ssh/id_ed25519.pub

# Once Niksa adds your key, connect:
ssh your-username@204.168.169.124

# Services running on the server:
# - n8n (automation): http://204.168.169.124:5678
# - Netdata (monitoring): http://204.168.169.124:19999

# You can also run Claude Code directly on the server:
ssh your-username@204.168.169.124
cd /opt/skd/trading-infra
claude
Phone access: Install Termius (iOS/Android) to SSH into the server from your phone. You can run Claude Code from anywhere.

! Windows Gotchas

Common issues and how to fix them on Windows.

Line Endings (CRLF vs LF)

Windows uses CRLF line endings, Linux uses LF. This causes git diffs to show every line as changed. Fix it:

# In WSL, set git to use LF
git config --global core.autocrlf input

# In Windows (PowerShell), set git to convert
git config --global core.autocrlf true

File Permissions

WSL and Windows handle file permissions differently. If you see permission errors:

# Fix SSH key permissions (must be done inside WSL)
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub

Path Differences

# Windows path:  C:\Users\YourName\Documents\skd
# WSL path:      ~/skd  (recommended — faster)
# WSL → Windows: /mnt/c/Users/YourName/Documents/skd (slower, avoid for code)

Obsidian on Windows + WSL Repos

Obsidian runs on Windows but your repos are in WSL. To access WSL files from Windows:

# In File Explorer, type this in the address bar:
\\wsl$\Ubuntu\home\YOUR_USERNAME\skd\brain

# Or map it as a network drive for easy access
Open Obsidian → Open folder as vault → navigate to \\wsl$\Ubuntu\home\YOUR_USERNAME\skd\brain
