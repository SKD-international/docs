How to Set Up Your Keys

Option A: Claude Max / Pro (Recommended)

If you have a Claude subscription, Claude Code authenticates through your browser — no API key needed.

# Just run Claude Code — it opens a browser to log in
claude

# Follow the authentication link
# Log in with your Anthropic account
# Done — Claude Code is connected to your subscription

Option B: API Key (Pay-as-you-go)

If you prefer pay-per-token, get an API key from the Anthropic Console.

# 1. Go to console.anthropic.com
# 2. Create an account & add payment method
# 3. Settings → API Keys → Create Key
# 4. Set a monthly spending limit (recommended: $20-50 to start)

# Set your API key (add to your shell profile):

# Mac/Linux (add to ~/.zshrc or ~/.bashrc):
export ANTHROPIC_API_KEY="sk-ant-your-key-here"

# Windows WSL (add to ~/.bashrc):
export ANTHROPIC_API_KEY="sk-ant-your-key-here"

# Then reload:
source ~/.bashrc

# Verify it works:
claude

Security Rules

  • X
    NEVER share your API key — not in Discord, not in Slack, not in code, not in git commits. If someone asks for your key, say no.
  • X
    NEVER commit API keys to git — use environment variables or .env files. Our repos have .gitignore that excludes .env files.
  • X
    NEVER use someone else's key — each person is responsible for their own usage and billing.
  • +
    DO set spending limits — on console.anthropic.com, set a monthly limit so you don't get surprised. Start with $20-50.
  • +
    DO rotate keys — if you think a key was exposed, delete it immediately and create a new one.
  • +
    DO use .env files — for any project that needs API keys, create a .env file (never committed to git).
# Example .env file (NEVER commit this)
ANTHROPIC_API_KEY=sk-ant-your-key-here
OANDA_API_KEY=your-oanda-key
OANDA_ACCOUNT_ID=your-account-id

# Load in Python:
from dotenv import load_dotenv
load_dotenv()

# Your .gitignore should contain:
.env
.env.local
.env.*.local

MCP Servers (Free)

MCP servers connect Claude Code to external services. They're free and open source — but they connect to your accounts.

MCP servers don't cost money, but they authenticate with your personal accounts (Slack, Gmail, GitHub, etc). Each person sets up their own MCP connections. You won't see Niksa's Slack messages — you'll see yours.

Each person configures their own MCP servers in ~/.claude/settings.json. See the Claude Code guide for setup instructions.

Shared Server (Hetzner VPS)

The VPS runs shared services. API keys for shared services are managed by admins only.

Admins only: OANDA API keys, Hetzner API token, and other production credentials are stored securely on the server. Members never need or see these keys.
